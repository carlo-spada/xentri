# Research Validation Checklist

> This checklist is used by the validate-research workflow to assess document quality.

## Document Structure

- [ ] Frontmatter present (Title, Date, Research Type)
- [ ] Research type clearly indicated (market, technical, competitive, etc.)
- [ ] Date of research documented
- [ ] No placeholder text remains
- [ ] Markdown headers properly structured

## Executive Summary

- [ ] Key findings summarized concisely
- [ ] Main conclusions stated upfront
- [ ] Scope of research described
- [ ] Methodology briefly explained (if applicable)

## Source Quality

### Citation Presence

- [ ] Key claims have citations
- [ ] Statistics include source references
- [ ] Quotes are properly attributed
- [ ] Data points are traceable

### Source Credibility

- [ ] Sources are credible and authoritative
- [ ] Primary sources used where possible
- [ ] Multiple sources for critical claims
- [ ] No reliance on unverified sources
- [ ] Industry/academic sources preferred over blogs

### Source Currency

- [ ] Source dates are documented
- [ ] Information is recent (within appropriate timeframe)
- [ ] Outdated sources clearly marked if included
- [ ] No reliance on stale data

### Link Validity

- [ ] Source URLs are accessible
- [ ] Links resolve correctly
- [ ] Archive links provided for volatile sources
- [ ] No broken references

## Content Accuracy

### Factual Accuracy

- [ ] Statistics are plausible
- [ ] Claims match cited sources
- [ ] No obvious contradictions
- [ ] Data is internally consistent
- [ ] Methodology is sound (if original research)

### Timeliness

- [ ] Information is current
- [ ] Market data is recent (< 12 months typically)
- [ ] Technology references are current
- [ ] Competitive landscape reflects current state

### Completeness

- [ ] Research scope is fully covered
- [ ] Key questions are addressed
- [ ] No obvious gaps in coverage
- [ ] Limitations are acknowledged

## Actionability

### Insights Quality

- [ ] Findings lead to clear implications
- [ ] Recommendations are specific and actionable
- [ ] Priorities are identified
- [ ] Risks and opportunities noted
- [ ] Next steps suggested

### Relevance

- [ ] Research addresses stated goals
- [ ] Insights are relevant to project context
- [ ] Information is usable for decision-making
- [ ] Connects to product/technical strategy

## Research Type Specific

### Market Research

- [ ] Market size quantified with sources
- [ ] Target segments defined
- [ ] Growth trends documented
- [ ] Competitive landscape covered

### Technical Research

- [ ] Technologies compared objectively
- [ ] Trade-offs documented
- [ ] Integration considerations noted
- [ ] Performance data included where relevant

### Competitive Research

- [ ] Key competitors identified
- [ ] Feature comparisons objective
- [ ] Pricing information sourced
- [ ] Differentiation opportunities noted

### User Research

- [ ] User segments clearly defined
- [ ] Pain points validated
- [ ] User goals documented
- [ ] Quotes/evidence from actual users (if applicable)

## Quality Assessment

### Critical Issues (unreliable research):

- [ ] None identified

### Warnings (use with caution):

- [ ] None identified

### Ready for Use:

- [ ] Yes, research is validated
- [ ] No, requires amendments (see issues above)
